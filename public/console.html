使用指南

快速开始 客户端接入 缓存标记 Prefix 锁定 缓存转译 渠道 高级配置 诊断 常见问题

快速开始

  1. 启动网关，确认控制台地址为 http://127.0.0.1:8788/console。
  2. 在酒馆 / 客户端里选择 OpenAI-compatible / Chat Completion 接入。
  3. Base URL 填 http://127.0.0.1:8788/v1，API Key 填你的上游供应商 Key。

酒馆里选择的是“入站格式”，决定客户端用哪种协议发给本地网关；控制台里的“上游格式”决定网关再用哪种协议转发给供应商。两者可以不同，例如酒馆用 OpenAI-compatible 接入，网关再按当前渠道设置转发到 OpenAI-compatible 或 Anthropic native 上游。

客户端接入

  • 本机使用：http://127.0.0.1:8788/v1
  • 不要把 API Key 写进渠道 Profile。Key 应放在客户端的 Authorization / x-api-key，或仅在私有机器上用 UPSTREAM_API_KEY 环境变量。

缓存标记

在稳定的大段内容末尾放置 [[CACHE_BREAK]]，网关会移除标记并注入 Claude prompt cache 的 cache_control。

  • 适合放在标记前：系统提示词、角色卡、蓝灯世界书、长设定、固定规则。
  • 适合放在标记后：绿灯世界书、最近聊天、当前输入、短期记忆、会频繁变化的内容。
  • Claude 每次请求最多支持 4 个缓存断点；多余标记会被移除但不会注入缓存控制。

Prefix 锁定 / 强制锁定

Prefix 锁定用于处理“前缀漂移”：如果世界书、正则、插件或数据库填表内容在缓存点前产生变化，缓存命中率会下降。

  • 开启后，第一个带缓存点的请求会教会网关稳定前缀。
  • 后续请求会用已学习的前缀替换当前请求的缓存点前内容，缓存点后的聊天、输入、绿灯世界书仍保持最新。
  • 它是替换，不是追加，不会把世界书、系统提示词或角色卡重复拼接。
  • 更换角色卡、世界书、预设或缓存点前内容后，请清空并重新学习。
  • 关闭“缓存转译”后，Prefix 锁定也会跳过。

缓存转译与 TTL

  • “缓存转译”开启时，网关会处理 [[CACHE_BREAK]]、注入 cache_control，并允许 Prefix 锁定参与。
  • 关闭后，网关不做缓存标记转译，也不会执行 Prefix 锁定；但渠道、高级配置、上游格式转换仍然生效。
  • TTL 选择 1h 会发送 Claude 1 小时缓存窗口；选择默认窗口则不发送 ttl，交给供应商默认处理。

渠道与供应商

  • 渠道 Profile 会保存名称、Base URL、上游格式和高级配置，不保存 API Key。
  • Pioneer 是默认自定义渠道；OpenRouter、Anthropic、Vertex、Bedrock 是内置模板。
  • OpenRouter 的“锁定供应商”会写入 provider 参数，可选择常见供应商，也可以填写自定义供应商名。
  • Vertex / Bedrock 卡片目前是 Profile 模板，不包含 Google Auth 或 AWS SigV4 签名；需要真实云鉴权时建议先走兼容供应商或自定义代理。

高级配置

  • 包含主体参数：把 JSON 对象深度合并到最终请求体，只在 OpenAI-compatible 上游格式生效。
  • 排除主体参数：按字段路径删除请求体字段，例如 stream_options 或 metadata.trace_id。
  • 包含请求头：添加 / 覆写非密钥 header，例如 HTTP-Referer、X-Title。
  • 排除请求头：发送上游前删除指定 header，例如 x-real-ip、x-forwarded-for。

高级配置按当前渠道保存。Authorization、x-api-key、Cookie、token、secret、password 等密钥类字段会被拒绝保存。

诊断

  • 请求诊断默认关闭，开启后只保存在内存里，重启即清空。
  • 诊断会记录最终请求体，可能包含私密提示词；不要公开导出的诊断 JSON。
  • 可通过诊断查看最终请求体、缓存断点、Prefix hash、suffix hash 和上游缓存用量。

常见问题

  • Mac 打不开 bat？ .bat 只适用于 Windows。macOS / Linux 请用 chmod +x start-gateway.sh 后运行 ./start-gateway.sh。
  • 请求 404 或路径不对？ 酒馆 Base URL 优先填写 http://127.0.0.1:8788/v1。
  • 请求日志要一直开吗？ 不要。请求诊断只在排查问题时临时开启；平常保持关闭即可。
  • 缓存没命中？ 确认蓝灯世界书在 [[CACHE_BREAK]] 之前、绿灯世界书在它之后；必要时开启 Prefix 锁定排查前缀漂移。
  • 使用数据库会影响缓存吗？ 数据库本身不影响缓存，官方渠道验证正常；若数据库蓝灯条目会随填表更新，请把全局注入位置改到缓存点后面，例如角色后、系统。
  • 上游不支持 1h TTL？ 在“缓存策略”里切回默认窗口。
